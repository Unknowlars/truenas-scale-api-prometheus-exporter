# app.container_log_follow

## Summary

Retrieve logs of a container/service in an app.

This event is generated by a dynamic source. If you want to specify subscription params, subscribe to it using the name in the following format `app.container_log_follow:{"param": "value"}`

## Required Roles

- `APPS_READ`

## Schema

- Type: object

### Subscription parameters

- Schema name: `AppContainerLogsFollowTailEventSourceArgs`
- Type: object
- No Additional Properties
#### tail_lines

- Schema name: `Tail Lines`
- Default: 500

Number of log lines to tail from the end of the log. If `null`, retrieve complete logs of the container.
##### Any of

###### Option 1

- Type: integer
- Value must be greater or equal to `1`

###### Option 2

- Type: null

#### app_name (required)

- Schema name: `App Name`
- Type: string

Name of the application whose container logs to follow.

#### container_id (required)

- Schema name: `Container Id`
- Type: string

Unique identifier of the specific container to get logs from.

### ADDED

- Schema name: `AppContainerLogsFollowTailEventSourceEvent`
- Type: object
- No Additional Properties
#### fields (required)

- Schema name: `AppContainerLogsFollowTailEventSourceEvent`
- Type: object
- No Additional Properties
##### data (required)

- Schema name: `Data`
- Type: string

The log line content.

##### timestamp (required)

- Schema name: `Timestamp`

Timestamp of the log entry. `null` if not available.

####### Option 1

- Type: string

####### Option 2

- Type: null
