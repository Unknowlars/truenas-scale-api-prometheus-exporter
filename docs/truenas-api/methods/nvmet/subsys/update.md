# nvmet.subsys.update

## Summary

Update NVMe target subsystem (`subsys`) of `id`.

## Required Roles

- `SHARING_NVME_TARGET_WRITE`

## Schema

- Type: object

### Call parameters

- Type: array
- No Additional Items
- Schema name: `Tuple Validation`

#### Parameter 1: id

#### id

- Schema name: `id`
- Type: integer

ID of the subsystem to update.

#### Parameter 2: nvmet_subsys_update

#### nvmet_subsys_update

- Schema name: `nvmet_subsys_update`
- Type: object

Updated configuration data for the NVMe-oF subsystem.
- No Additional Properties
##### name

- Schema name: `Name`
- Type: string

Human readable name for the subsystem. If `subnqn` is not provided on creation, then this name will be appended to the `basenqn` from `nvmet.global.config` to generate a subnqn.
- Must be at least `1` characters long

##### subnqn

- Schema name: `Subnqn`

NVMe Qualified Name (NQN) for the subsystem. Must be a valid NQN format if provided.
###### Any of

####### Option 1

- Type: string
- Must be at least `11` characters long
- Must be at most `223` characters long

####### Option 2

- Type: null
